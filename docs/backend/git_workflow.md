# Flujo Git (Fase 0)

## Ramas

- `main`: versión estable.
- `develop`: integración.
- `feature/be-0-x-nombre`: trabajo por tarjeta.

## Commits

Usar Conventional Commits:

- `feat: agrega modelo animal`
- `fix: corrige conexion postgres`
- `docs: agrega arquitectura fase 0`

## Pull Request

Todo cambio entra por PR hacia `develop` con revisión de TL.
